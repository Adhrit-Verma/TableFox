# Product

## Register

product

## Users
Database engineers, backend developers, data platform owners, and AI agents that need to understand a PostgreSQL database quickly without manual schema spelunking. Humans use it locally while exploring unfamiliar databases; agents use the MCP tools to navigate schema structure and relationships incrementally.

## Product Purpose
Database Agent turns PostgreSQL metadata into a stable graph map that both people and LLMs can search, traverse, and explain. Success means users can identify important tables, relationship paths, columns, and constraints quickly while using read-only credentials by default.

## Brand Personality
Precise, calm, technical. The interface should feel like an expert instrument: information-dense, legible, restrained, and fast.

## Anti-references
Avoid marketing-style SaaS landing pages, decorative dashboards, oversized hero sections, gamified database visuals, glassy cards, and bright neon graph tools that make schema work feel less trustworthy.

## Design Principles
- Make the database structure the primary visual object.
- Keep navigation incremental so large schemas stay understandable.
- Prefer read-only safety and explicit state over hidden automation.
- Use dense controls when they reduce time to insight.
- Let agents and humans share the same graph model and stable IDs.

## Accessibility & Inclusion
Target WCAG AA contrast, visible focus states, keyboard-accessible controls, reduced-motion support, and status text that does not rely on color alone.
